GradeVault Guide

System Walkthrough & User Journey

Overview Phase 1: The Setup Phase 2: Student Exp. Phase 3: Grading Day Technical Stack

System Overview

KLD GradeVault streamlines the grading process for a multi-institute college by digitizing grade submission, verification, and distribution.

Hierarchy-Aware

Respects organizational structure. Institute Heads only manage their own faculty.

Hybrid Enrollment

Students join via "Class Codes" or are Auto-Enrolled when a teacher uploads a grade sheet.

Data Integrity

Grades are calculated automatically (Raw Score $\to$ Transmuted Grade) to prevent errors.

Phase 1: The Setup (Admin & Teacher)

Step 1: Teacher Registration

Account Creation

File: register.php

1 User registers with an institutional email (@kld.edu.ph).
2 Selects Institute (e.g., "Institute of Nursing") and Role ("Teacher").
3 System sets status to Pending Approval.

Step 2: Admin Approval

Faculty Verification

File: admin_faculty.php

1 Institute Head logs in and sees "Pending Approvals".
2 Reviews details and clicks [Approve].
3 Teacher account becomes Active.

Step 3: Creating a Class

Class Management

File: my_classes.php

1 Teacher logs in and clicks "Create Class".
2 Inputs Subject (e.g., ANAT101), Section, and Semester.
3 System generates a unique Class Code (e.g., G7K-9L2).

Phase 2: The Student Experience

Step 4: Student Registration

Instant Access

File: register.php

Students register with their school email. Unlike teachers, they get Instant Access without admin approval.

Step 5: Joining a Class

Enrollment

File: student_classes.php

1 Student logs in and clicks "Join Class".
2 Enters the 6-digit Class Code provided by the teacher.
3 The subject now appears on their dashboard.

Phase 3: Grading Day

Step 6: Uploading Grades

Excel Import & Transmutation

File: grade_upload.php

1 Teacher selects a class and uploads an Excel file with Raw Grades (0-100).
2 Auto-Calculation: System converts Raw Grades to Transmuted Grades (e.g., 95 $\to$ 1.25).
3 Auto-Enrollment: If a student in the file is not in the class, the system automatically enrolls them ("Ghost Student Fix").

Transmutation Logic:
97-100 $\to$ 1.00 | 94-96 $\to$ 1.25 | ... | < 75 $\to$ 5.00 (Failed)
Step 7: Viewing Results

Grade Monitoring

File: student_history.php

Student logs in to see their new grades. They can view a detailed history grouped by semester and download an official grade report.

Technical Stack

Frontend

  • Bootstrap 5: Responsive UI
  • SheetJS: Excel parsing in browser
  • Verdant Design System: Custom CSS

Backend

  • PHP 8+: Core logic
  • MySQL: Relational database
  • RBAC: Role-Based Access Control
